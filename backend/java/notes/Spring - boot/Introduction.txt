Spring Boot Introduction:
-------------------------
Webapp - it has the capability to run on server. And it can able to take the requests, process it and respond with response. 
If we develop webapps using Spring we get below issues:
     -> Manually we have to configure the server.
     -> Add Many dependencies(jars) one by one.
     -> Handling version compatabilities.
     -> Build app to a war file, then deploy(copy the war file.) the war file in the server 

Spring Boot solves all the above problem.
     -> Auto Configurations  / Convention Over Configuration.(You write business logic, all the configuration will be given by spring boot.)
           Instead of asking to configure everything in manual. it assumes default configurations and sets them automatcally.
         Before Spring Boot(using Spring web.) the below are problems
            -> Server Setup
            -> Map URL to classes and methods.   
            -> Component scanning(Spring context, spring AOP, spring core)
            -> JSON Converters.
         After Spring boot all the above problems are taken care :
             - Embeded Server( EX: Apache Tomcat or Jetty)
             - REST Support
             - JSON Converters, we will get by default in spring boot.
             - Request and Response handling are take care by spring boot.
             - Default Port assignment by spring boot.
     -> Starter Dependencies.
         Starter dependencies are predefined budles of commonly used libraries.
         EX: If i want to develop web app, i dont need to add all the depdencies instead i can just user one starter depency which will configure all we app related dependency.
            -> spring-boot-starter-web - It will fetch all the dependencies related to web and attaches to the project.
            -> spring-boot-starter-data--jpa - It will fetch all the dependencies related to data jpa(spring tx, spring ORM, spring data etc dependencies) and attaches to the project.
         this will handle Version cpmpatability.
              It ensures included libs inside the starter dependencies work well together, so we can avoid dependency conflicts.
     -> Easy Production setup.
          Production ready app (Once the app is developed the application is ready and to be served in the production for real users.)
          App Monitoring and health checks can be done using Spring actutors.
            -> Using spring boot actuators we can 
                 -> monitor health
                 -> memomory usage of app.  

           External Configuration
            It will support external configuration using application properties.
               EX: for each environment(EX: Development, test, Production )  we have to configure databases seperately. those configs are 
               written in application.properties for environmet wise.
