1. Synchronous Programming
=>  the code will be executed line by line, from top to bottom. 

Ex:
// synchronous code example
console.log("hello");
console.log("welcome to js");
console.log("learn js");

Problems in Synchronous Programming:
=> If a task takes a long time (like downloading a huge file), the entire program "freezes" and waits for that task to finish. 
Nothing else can happen until that line is done.

Real-Life Example: The Coffee Shop
Imagine you are at a coffee shop with only one barista and a very strict rule:

=> Customer A orders a coffee.
=> The Barista starts making the coffee.
=> Customer B stands at the counter and must wait until Customer A's coffee is finished and handed over before they can even place their order.
=> If the coffee machine breaks and takes 10 minutes to fix, Customer B just stands there waiting.
=> This is a Synchronous process. It’s simple, but it can be slow if one task takes a long time.


Pros:
Easy to read: The code looks like a story (Start → Middle → End).
Easy to debug: You know exactly which line is running at any time.

Cons:
Blocking: If one line is slow (like loading a heavy image), the whole website freezes.
Bad User Experience: Users hate it when buttons stop responding because the code is "busy."


----------------------------------------------------------------------------------------------------------


2. Asynchronous Programming:
Asynchronous programming allows the computer to start a long task, move it to the "background," and continue running the rest of the code.
When the long task is finally finished, the computer brings the result back.

Ex:
console.log("Step 1: I am hungry. I ordered a pizza.");
// This task happens in the background (takes 3 seconds)
setTimeout(() => {
    console.log("Step 2: The pizza is ready!");
}, 3000);
console.log("Step 3: While waiting, I am watching TV.");

Real Life Example:
Imagine a restaurant with a waiter and a chef:

=> Customer A orders a Pizza.
=> The Waiter takes the order to the kitchen.
=> Crucial Part: The waiter does not stand at the kitchen door waiting for the pizza to cook. Instead, the waiter goes to Customer B to take their order.
=> When the Pizza is ready, the chef rings a bell, and the waiter goes back to pick it up and serve it.
=> This is Asynchronous. Multiple things (ordering, cooking, serving) are happening at the same time without anyone being "blocked."

Real Time Usage:-
On a real website, we use Asynchronous programming for:
=> Fetching data from a server (API calls).
=> Uploading files (so the user can still use the app while it uploads).
=> Timers or animations.
