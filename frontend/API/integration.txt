1. what is API integration ?
=> the process of connecting two or more applications via their APIs to exchange data and perform actions automatically.
=> An API is the "doorway" or the set of rules, Integration is the actual act of "walking through that door" to make two systems work together as one.

why it is needed ?
When you integrate an API, you are creating a permanent "handshake" between systems. Here is a standard workflow:
=> Authentication: Your application proves its identity to the other system (usually using an API Key or OAuth token).
=> The Request: Your app sends a specific command (e.g., "Give me the latest stock prices").
=> The Processing: The external server processes the request.
=> The Payload: The external server sends back the data (usually in JSON format).
=> The Action: Your app takes that data and displays it or uses it to trigger another process.

2. Fetch()
=> It is a modern, built-in browser method used to make network requests.
=> It allows you to retrieve data from a server (like a list of users or weather info) or send data to a server (like a login form) without refreshing the page.
=> The most important thing to know about fetch() is that it is asynchronous.
=> When you call fetch(), it returns a Promise.

Response after API call:
=> A Promise will be returned and it is like a "IOU" note. 
=> It says: "I don't have the data right now, but I promise to give it to you once the server responds."

API Integration example:

// html code
<button onclick="getData()" type="button">Get Data </button>

function getData() {
    fetch('https://dummyjson.com/users')

        // first part => response from server as promise
        .then(res => {
            if (res.ok) {
                console.log("response", res);
                return res.json();
            }
        })

        // second part => promise data converted to json format
        .then(data => {
            console.log("data", data);
        })

        // third part => error handling
        .catch(err => {
            console.log("error", err);

        })
}

Problems in above approach:
1. "Callback Hell" (Pyramid of Doom)
=> If you need to make three API calls in a row (e.g., Get User → Get User's Posts → Get Post Comments), your code will keep nesting deeper and deeper to the right. 
=> This makes the code very hard to read and maintain.

2. Scoping Issues
=> Variables declared inside one .then() block are not accessible in the next block. 

3. Error Handling Fragmentation
=> In your current code, if the res.json() fails, or if there is a logic error in the second .then(), the .catch() might handle it, but it’s often hard to pinpoint where the error happened. 

4. Readability
=> Humans naturally read code from top to bottom. Promises force the brain to read code as a "chain of events," which is less intuitive than the synchronous-looking style of async/await.


----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


3. Solution: async/await with try/catch
=> The modern solution is to use async/await.
=> It makes asynchronous code look and behave like synchronous code, making it much easier to debug and read.


Ex:
<button onclick="getData()" type="button">Get Data </button>

async function getData() {
    try {
        // 1. Wait for the server response
        const response = await fetch('https://dummyjson.com/users',{
            method:"GET"
        });

        // 2. Immediate check for HTTP errors (404, 500, etc.)
        if (!response.ok) {
            throw new Error(`HTTP error! status: ${response.status}`);
        }

        // 3. Wait for the data to parse into JSON
        const data = await response.json();
        
        // 4. Use the data
        console.log("Success Data:", data);

    } catch (err) {
        // 5. One central place to catch ANY error (Network, Syntax, or HTTP)
        console.error("Fetch failed:", err.message);
    }
}
