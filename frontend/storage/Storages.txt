WHAT IS BROWSER STORAGE?
Browser storage is used to:
1. Store data in the user’s browser
2. So that data can be used later (even after refresh)

Why we use it?
Save user data (name, theme, cart, etc.)
Avoid calling backend again
Improve user experience

TYPES OF BROWSER STORAGE
1️.Local Storage
2️.Session Storage

-----------------------------------------------------------------------------------------------------

1. LOCAL STORAGE
Local Storage is a browser storage that:
a) Stores data permanently in the browser
b) Data remains even after refresh or closing the browser


Key Features:
No expiry (until manually deleted)
Stores data as key–value pairs
Size ~5MB
Same origin only (same website)


METHODS OF LOCAL STORAGE

1. setItem(key, value)
setItem is a built-in javascript method used to store the data in the form of string.

Syntax:
localStorage.setItem("name", "naveen");




2. getItem(key)
getItem is a method used to get stored data from storge using key as reference.

Syntax:
let name = localStorage.getItem("name");
console.log(name);
O/P: naveen


3. removeItem(key)
this method will removes particular data using key as reference.

Syntax:
localStorage.removeItem("name");


4. clear()
Removes all stored data

Syntax:
localStorage.clear();


5. key(index number)
It will return key using the index number of the storage data.

Syntax:
localStorage.key(2);

O/P:
It will return key for that particular index number from storage

REAL-TIME EXAMPLES (LOCAL STORAGE)
1. Remember Username
User enters name → saved
Next time → auto-filled

2. Dark Mode/Light Mode Toggle
localStorage.setItem("theme", "dark");
When page loads:
if(localStorage.getItem("theme") === "dark"){
  document.body.style.background = "black";
}

3. Shopping Cart (Myntra application)
Store products added to cart
Even after closing tab again we can see our added cart item


---------------------------------------------------------------------------------------------------------

2. SESSION Storage

Session Storage is a browser storage that:
a-> Stores data only for a single browser tab/session
b-> Data is deleted when the tab is closed

Key Features:
Temporary storage
Cleared when tab closes
Same methods as localStorage


METHODS OF SESSION STORAGE

1. setItem(key, value)
setItem is a built-in javascript method used to store the data in the form of string.

Syntax:
sessionStorage.setItem("name", "naveen");


2. getItem(key)
getItem is a method used to get stored data from storge using key as reference.

Syntax:
let name = sessionStorage.getItem("name");
console.log(name);
O/P: naveen


3. removeItem(key)
this method will removes particular data using key as reference.

Syntax:
sessionStorage.removeItem("name");


4. clear()
Removes all stored data

Syntax:
sessionStorage.clear();


5. key(index number)
It will return key using the index number of the storage data.

Syntax:
sessionStorage.key(2);

O/P:
It will return key for that particular index number from storage

REAL-TIME EXAMPLES (SESSION STORAGE)
1. Temporary Login Data
Store user info during session
Logout or close tab → data cleared

2. Multi-step Form
Step 1 → save data
Step 2 → retrieve data
Close tab → reset

3. OTP Verification
Store OTP temporarily
Auto delete after session


LOCAL vs SESSION STORAGE:
| Feature  | Local Storage  | Session Storage  |
| -------- | -------------- | ---------------- |
| Lifetime | Permanent      | Until tab closes |
| Storage  | Yes            | Yes              |
| Use Case | Long-term data | Temporary data   |
